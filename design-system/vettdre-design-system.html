A comprehensive, production-ready design language for the AI-native real estate platform. Built for NYC real estate professionals who demand precision, clarity, and intelligence.

Design Principles

Six core principles guide every design decision across VettdRE. They ensure consistency, usability, and a premium experience befitting NYC's most demanding real estate professionals.

1. Intelligence Forward

AI insights should surface naturally within the workflow, not hide behind extra clicks. Data-rich interfaces earn trust through transparency — show confidence scores, source attribution, and freshness timestamps. Every AI action must be explainable.

2. Precision Over Decoration

Real estate decisions involve millions of dollars. Every pixel must serve a purpose. Favor clear data hierarchy, scannable layouts, and decisive typography over ornamental flourish. Remove anything that doesn't help the user make a better decision faster.

3. Progressive Disclosure

Show the essential information first, then reveal depth on demand. Building profiles start with key metrics; detail tabs expand into violations, permits, and ownership chains. Never overwhelm — always invite exploration.

4. Contextual Density

Professional users want information density — but calibrated to context. Dashboards can be data-rich; onboarding must be spacious. Mobile demands ruthless prioritization. The same data may need three different presentations across three contexts.

5. Trust Through Consistency

Every interaction pattern, every color meaning, every spacing decision must be predictable. Green always means positive. Navy always anchors primary actions. The 8px grid is sacred. Consistency compounds into trust.

6. Speed as a Feature

Performance is a design choice. Skeleton loaders appear within 100ms. Progressive rendering shows data as it arrives. Optimistic updates make interactions feel instant. Perceived speed is as important as actual speed.

Do's & Don'ts

Clear guidance for consistent implementation across the platform.

Color

Use semantic colors for meaning

Green for success/closed, red for errors/overdue, amber for warnings. Always pair color with icon or text label.

Use color as sole indicator

Never rely only on color to convey status. 8% of males have color vision deficiency. Always add text or iconography.

Typography

Use the type scale consistently

Stick to the 12 defined levels. DM Serif Display for display headings only; Inter for everything else.

Invent custom font sizes

Never use arbitrary sizes like 17px or 22px. If the scale doesn't have what you need, discuss adding a new level.

Spacing

Respect the 8px grid

All spacing values must be multiples of 4px. Primary rhythm is 8px. Use tokens (space-2, space-4, space-6, etc.).

Use arbitrary padding or margins

Never use values like padding: 13px or margin: 7px. If it doesn't align to the grid, it will look inconsistent.

Components

Use existing components

Check the design system first. If a component exists that solves your need, use it. Extend with variants if needed.

Build one-off custom components

Don't create a new button variant or card style for a single use case. If you need something new, propose it as a system addition.

Data Display

Use JetBrains Mono for numbers

Monetary values, BBLs, percentages, dates in tables, and IDs all use the monospace font for alignment and readability.

Mix fonts in data contexts

Don't display $4,250,000 in Inter next to 12.4% in JetBrains Mono. Be consistent within a data context.

Mobile

Design mobile-first, then expand

Start with the mobile layout. Use pb-safe for notch devices. Bottom sheets replace modals. Bottom tab bar for nav.

Simply shrink the desktop layout

Mobile is not a smaller desktop. Rethink information priority, collapse multi-pane to single-pane, use touch-appropriate targets.

AI Content

Always indicate AI-generated content

Mark AI outputs with the sparkle icon and "AI" badge. Show confidence scores. List data sources with freshness timestamps.

Present AI content as absolute truth

Never hide that content is AI-generated. Never show 100% confidence. Always provide a way for users to edit or override AI suggestions.
